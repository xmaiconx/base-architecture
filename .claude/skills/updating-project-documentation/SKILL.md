# Updating Documentation

Atualize o documento com uma descrição de até 50 palavras sobre a funcionalidade e inclua caminho dos arquivos mais importantes com regras de negócio, sem exemplos de códigos, com uma breve descrição de ~10 palavras sobre cada arquivo ou grupo de arquivos baseado nas implementações realizadas.

Caso não haja implementação no seu histórico e a branch for uma branch de desenvolvimento (feat, fix e etc - **sem ser master ou production**), pergunte para o usuário se deseja que analise a implementação realizada na branch para realizar a atualiação da documentação solicitada.
